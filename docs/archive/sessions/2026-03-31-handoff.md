## Key Decisions

### Voice = transport, not a separate system
Voice chat uses the **same agent, same kernel, same session, same tools** as text chat. `kernel.send_message_streaming()` doesn't change. Voice is a presentation layer that sits between the WebSocket and the kernel.

### No WebRTC, no Google, no cloud APIs
- **Transport:** Binary WebSocket + Opus over Tailscale (iPhone is on the mesh)
- **STT:** Local — Faster Whisper (port 7733) or LFM2.5-Audio ASR mode
- **TTS:** Local — Kokoro (port 7744) or LFM2.5-Audio TTS mode
- **iOS client:** Native Apple frameworks only — AVAudioEngine, URLSessionWebSocketTask, libopus static lib. No pods, no Google.

### LFM2.5-Audio-1.5B (Liquid AI) is being evaluated
- Self-hostable, 1.5B params, English-only
- Supports separate ASR and TTS modes via system prompts
- Supports native speech-to-speech via `generate_interleaved()`
- No built-in VAD or turn detection — app-layer responsibility
- Best Rust inference path: `ort` crate with official ONNX weights (Q4, ~1.5GB)
- Cannot run on Candle (zero LFM architecture support, 6-10 week effort)
- Not yet tested on prtr hardware

### User rejects ElevenLabs pipeline paradigm
The sentence-split → TTS approach was explicitly rejected. User wants "advanced voice" feel — natural turn-taking, not robotic text-reading. The voice layer should do lightweight markdown cleanup on `StreamEvent::TextDelta`, not complex sentence boundary detection. Audio cues for tool use / thinking, not silence.

## Architecture (final)

```
ipjsua-swift (reworked iOS app)
  AudioManager.swift (unchanged)
  + libopus static lib
  + OpenFangVoiceClient (new, replaces ViceClient)
      ↕ Binary WS over Tailscale
openfang-api/ws.rs
  + binary frame handling
  + voice session state machine
      ↕ text
kernel.send_message_streaming() ← unchanged
      ↕ StreamEvent channel
voice transport layer (new, in ws.rs or openfang-rtc crate)
  + StreamEvent filtering (TextDelta → speak, Tool*/Thinking → skip or audio cue)
  + markdown-to-speech cleanup
  + VAD for turn detection
  + TTS via LFM2.5-Audio or Kokoro
  + STT via LFM2.5-Audio or Faster Whisper
```
