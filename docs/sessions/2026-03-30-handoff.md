## Root cause: Z.AI vs Zhipu account confusion

The original integration treated Z.AI and Zhipu as the same account (same API key, very similar endpoint). They are different:

| | Zhipu | Z.AI (Coding/Devpack) |
|---|---|---|
| Base URL | `open.bigmodel.cn/api/paas/v4` | `api.z.ai/api/coding/paas/v4` |
| API key env | `ZHIPU_API_KEY` | `Z_AI_API_KEY` |
| Model IDs | `glm-5-20250605`, `glm-4v`, … | `glm-5.1`, `glm-5`, `glm-5-turbo`, `glm-4.7` |
| Billing | Pay-per-token | Devpack subscription |

The `/models` endpoint on `api.z.ai/api/coding/paas/v4` does not return a model list (returns empty or 404), which is why the dashboard's "Add Custom Provider" flow could not auto-populate models. Models must be specified by name explicitly.
