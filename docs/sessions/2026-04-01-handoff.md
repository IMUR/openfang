## Completed This Session

- **`query_graph` SurrealQL parse error fixed** (`crates/openfang-memory/src/knowledge.rs`)
  Two illegal constructs were causing every knowledge graph query to fail:
  1. `FROM relations r` — `relations` is a reserved word in SurrealDB; now backtick-quoted
  2. `LET s = r.in, t = r.out` after `FROM` — `LET` is a standalone statement in SurrealDB,
     not an inline alias clause
  Rewrote to use native `in.*`/`out.*` record-link expansion and an object constructor for the
  `relation` sub-object. WHERE conditions updated (`s.name` → `in.name`, `r.relation_type` →
  `relation_type`, etc.). `traverse_from` multi-hop paths (`->relations->entities`) were
  unaffected — those use the native graph traversal operator.

- **Rebuilt and deployed with `--features memory-candle`**
  Previous April 1 build omitted the flag; all four ML subsystems were compiled out.
  Rebuilt (~17min), deployed via `systemctl --user stop/copy/start`.
  Health confirmed: `memory_candle_binary: true`, `embedding_active: true`, `ner_active: true`,
  `reranker_active: true`, `models_cached: true`.
