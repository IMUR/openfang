## Discovered: `config.rs` Pollution

`crates/openfang-types/src/config.rs` is 4,571 lines containing ~120 structs.
It is functioning as a catch-all for every config type in the system rather
than a shared types crate.

**What's in there that shouldn't be:**

- **40+ channel adapter configs** (~2,000 lines) — `TelegramConfig`,
  `DiscordConfig`, `SlackConfig`, and 37 others. These belong in
  `openfang-channels`, not in the types crate that every other crate
  depends on. Every channel struct change forces a full workspace
  recompile.
- **`TtsConfig`, `TtsOpenAiConfig`, `TtsElevenLabsConfig`** — a dead TTS
  abstraction layer predating the current Python voice services. No code
  in the active voice pipeline uses these; they're stale upstream API
  surface that never applied to this deployment.
- **`VoiceConfig` defaults** — the Rust struct defaults for `stt_model`
  (`distil-large-v3`) and `tts_voice` (`af_heart`) are inherited from
  the upstream Whisper + Kokoro era. They compile silently but produce
  a broken voice pipeline on any deployment that hasn't explicitly
  overridden them.

**What's fine:**

- `KernelConfig`, `MemoryConfig`, `VoiceConfig` (struct itself, not defaults),
  `WebConfig`, `BrowserConfig`, `ExecPolicy`, `BudgetConfig`, `AuthConfig`,
  `McpServerConfigEntry` — these are legitimately shared kernel config.

**Impact of current state:**

- Incremental build times are inflated for any channel adapter change.
- Dead TTS structs (`TtsConfig` etc.) create false surface area in the API.
- `VoiceConfig::default()` is actively dangerous to anything that
  constructs it without explicitly setting the four required fields.

## Open Items

### High — `VoiceConfig::default()` stale values

`stt_model = "distil-large-v3"` and `tts_voice = "af_heart"` in
`VoiceConfig::default()` should either be updated to neutral placeholder
strings that obviously require overriding (e.g. empty strings or
`"<set me>"`) or the field should be `Option<String>` so the absence is
explicit.

**Resume point:** `crates/openfang-types/src/config.rs` lines 606–620,
`VoiceConfig::default()`.

**Minimal fix:**
```rust
stt_model: String::new(),   // was "distil-large-v3" (Whisper — retired)
tts_voice: String::new(),   // was "af_heart" (Kokoro — retired)
```
Also update the doc comments on lines 582 and 584 to remove the
now-incorrect default values.

### Medium — `config.rs` channel adapter split

The 40+ channel adapter configs should move to
`crates/openfang-channels/src/config.rs` (or a `config/` module there).
`KernelConfig` can keep a `channels: ChannelsConfig` field that the
channels crate owns. This is a non-trivial refactor but the path is
mechanical — no logic changes, just moving struct definitions and updating
`use` paths.

**Not urgent.** The build works. But every time a channel adapter is
touched, the whole workspace recompiles from `openfang-types`. This will
compound as the channel list grows.

### Medium — `TtsConfig` / `TtsOpenAiConfig` / `TtsElevenLabsConfig` removal

These structs appear to be upstream surface for a different TTS
integration approach. Verify they're unreferenced in the active codebase
before deleting:

```bash
rg 'TtsConfig|TtsOpenAiConfig|TtsElevenLabsConfig' crates/ --type rust
```

If the only references are in `config.rs` itself and `KernelConfig`'s
field declarations, they can be removed cleanly.
