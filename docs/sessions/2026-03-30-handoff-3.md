## Key decisions

| Decision | Rationale |
|----------|-----------|
| `broadcast::Sender` in ApprovalManager, not EventBus | Avoids kernel init reordering; EventBus is Arc-less and constructed separately |
| WebSocket, not SSE | Chat uses WebSocket; SSE is for comms/logs. Aligned to existing transport |
| No SurrealDB for approvals | Pending approvals are in-process memory (DashMap + oneshot). Surreal not needed for v1 |
| Single fan-out channel with filtering | Simpler than per-agent channel map; sufficient for single-process |

## What went wrong

The build/deploy step consumed most of the session (~2 hours of debugging) because:

1. Cursor's sandbox sets `CARGO_TARGET_DIR` to a temp directory under `/tmp/cursor-sandbox-cache/`
2. Every `cargo build` succeeded and produced a correct binary — in that temp directory
3. The agent kept inspecting `target/release/openfang` (an old binary from a pre-sandbox build)
4. `strings` checks, timestamp analysis, fingerprint debugging — all red herrings
5. Solution was `echo $CARGO_TARGET_DIR` — one command at any point would have resolved it

**Lesson for future agents:** When Cargo reports success but the expected binary doesn't appear at `target/release/`, check `CARGO_TARGET_DIR` and `.cargo/config.toml` FIRST. Do not chase timestamps or fingerprints.

**Secondary lesson:** Previous session deployed to `~/.openfang/openfang`. The rtr-openfang skill documents `~/.local/bin/openfang` as the correct path. Use the skill.
